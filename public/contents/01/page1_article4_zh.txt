## “已超出配额”错误的解决办法
ㅤ
使用 API 时，是否突然看到 [[purple:"已超出配额"]] 的错误提示？
别慌。
这是非常常见的问题，而且有完美的解决方案。
ㅤ
## 先了解当前状况
你现在的 API 密钥处于 24 小时等待状态。但并不一定要干等着！   
ㅤ
[IMAGE:04_image_2.png]

## 解决方案 1.
ㅤ
这是最简单且最有效的方法。
Google 对注册限制不高。
因此你可以 [[purple:创建多个 Google 账号，并分别申请 API 密钥]]。
ㅤ
## 实战用法
1. 平时就用[[purple:3–5 个 Google 账号先申请好 API 密钥]]备用
2. [[purple:第一个密钥触顶后切换到第二个密钥]]
3. 采用轮换系统依次使用
ㅤ
## 先尽可能多地准备 API。
ㅤ
API 密钥 1 (main.account@gmail.com) - ABCDEFG...
API 密钥 2 (backup.account@gmail.com) - ABCDEFG...
API 密钥 3 (extra.account@gmail.com) - ABCDEFG...
像这样保存密钥，并进行轮换。

[IMAGE:04_image_3.png]

这样就可以无限使用。
像下图那样记在备忘里，然后按顺序轮换。
API 越多越好。

## 解决方案 2.

调整 API 的配额使用同样是重要策略。
只需缩小搜索范围，就能大幅节省配额。

[IMAGE:04_image_5.png]
ㅤ
[[purple:低效做法（应避免）]]
- 面向所有国家进行搜索
- 设置过于宽泛的范围
- 一次性请求过多数据
ㅤ
[[purple:高效做法（推荐）]]
- 只选择所需国家（如仅限韩国、美国、日本等目标国家）
- 按地区依次搜索
- 只请求真正需要的数据
ㅤ
