## 「割り当て量を超えました」エラーの解決法
ㅤ
API を使っていて、突然 [[purple:"割り当て量を超えました"]] というエラーメッセージを見ましたか？
慌てないでください。
これは非常によくある問題で、完璧な解決策があります。
ㅤ
## 現在の状況を把握
現在、あなたの API キーは 24時間の待機状態です。 しかし、ただ待つ必要はありません！   
ㅤ
[IMAGE:04_image_2.png]

## 解決 1.
ㅤ
最もシンプルで効果的な方法です。
Google は会員登録に大きな制限を設けていません。
したがって、[[purple:複数の Google アカウントを作成し、それぞれで API キー]]を発行できます。
ㅤ
## 実戦活用
1. ふだんから[[purple:3〜5個の Google アカウントで API キー]]を発行して準備
2. [[purple:最初のキーが上限に達したら2番目のキー]]に切り替え
3. ローテーションシステムで順番に使用
ㅤ
## まずは API をできるだけ多く確保しましょう。
ㅤ
API キー 1 (main.account@gmail.com) - ABCDEFG...
API キー 2 (backup.account@gmail.com) - ABCDEFG...
API キー 3 (extra.account@gmail.com) - ABCDEFG...
このようにキーを保存して、ローテーションさせてください。

[IMAGE:04_image_3.png]

こうすれば無制限に使えます。
下の画像のようにメモ帳に書いて、ローテーションさせてください。
API は多ければ多いほど良いです。

## 解決 2.

API の割り当て量の使い方を調整することも重要な戦略です。
検索範囲を絞るだけで、割り当て量を大きく節約できます。

[IMAGE:04_image_5.png]
ㅤ
[[purple:非効率な方法（避けるべき）]]
- 全ての国を対象に検索
- 不必要に広い範囲の設定
- 一度に大量のデータを要求
ㅤ
[[purple:効率的な方法（推奨）]]
- 必要な国だけを選択（韓国、米国、日本などのターゲット国のみ）
- 地域ごとに順番に検索
- 本当に必要なデータだけを要求
ㅤ
