## Решение ошибки «Квота превышена»
ㅤ
При работе с API вы вдруг увидели сообщение об ошибке [[purple:"Квота превышена"]]?
Не паникуйте.
Это очень частая ситуация, и есть идеальное решение.
ㅤ
## Оцените текущую ситуацию
Ваш ключ API сейчас находится в 24-часовом режиме ожидания. Но не обязательно просто ждать!   
ㅤ
[IMAGE:04_image_2.png]

## Решение 1.
ㅤ
Это самый простой и эффективный способ.
У Google мало ограничений на регистрацию.
Поэтому вы можете [[purple:создать несколько аккаунтов Google и выпустить ключ API для каждого]].
ㅤ
## Практическое применение.
1. Заранее [[purple:подготовьте ключи API в 3–5 аккаунтах Google]]
2. [[purple:Когда первый ключ достигнет лимита, переключайтесь на второй]]
3. Используйте по кругу в системе ротации
ㅤ
## Сначала обеспечьте как можно больше ключей API.
ㅤ
Ключ API 1 (main.account@gmail.com) - ABCDEFG...
Ключ API 2 (backup.account@gmail.com) - ABCDEFG...
Ключ API 3 (extra.account@gmail.com) - ABCDEFG...
Сохраните ключи вот так и используйте их по очереди.

[IMAGE:04_image_3.png]

Так вы сможете пользоваться без ограничений.
Запишите их в блокнот, как на картинке ниже, и чередуйте.
Чем больше ключей API, тем лучше.

## Решение 2.

Управление расходом квоты API — тоже важная стратегия.
Достаточно сузить область поиска — и вы серьёзно сэкономите квоту.

[IMAGE:04_image_5.png]
ㅤ
[[purple:Неэффективный подход (избегать)]]
- Поиск по всем странам
- Чрезмерно широкий охват
- Слишком большие запросы за один раз
ㅤ
[[purple:Эффективный подход (рекомендуется)]]
- Выбирайте только нужные страны (только целевые: Корея, США, Япония и т. п.)
- Ищите последовательно по регионам
- Запрашивайте только действительно необходимые данные
ㅤ
