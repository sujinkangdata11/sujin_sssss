## Solução para o erro “Cota excedida”
ㅤ
Usando a API, você viu de repente a mensagem de erro [[purple:"Cota excedida"]]?
Não entre em pânico.
Isso é muito comum e existe uma solução perfeita.
ㅤ
## Entenda a situação atual
Sua chave de API está atualmente em estado de espera de 24 horas. Mas você não precisa apenas esperar!   
ㅤ
[IMAGE:04_image_2.png]

## Solução 1.
ㅤ
Este é o método mais simples e eficaz.
O Google impõe poucas restrições de cadastro.
Portanto, você pode [[purple:criar várias contas do Google e emitir uma chave de API para cada uma]].
ㅤ
## Uso prático.
1. [[purple:Emita previamente chaves de API com 3–5 contas do Google]] e deixe-as prontas
2. [[purple:Quando a primeira chave atingir o limite, troque para a segunda]]
3. Use-as em rodízio com um sistema de rotação
ㅤ
## Primeiro, garanta o máximo possível de APIs.
ㅤ
Chave de API 1 (main.account@gmail.com) - ABCDEFG...
Chave de API 2 (backup.account@gmail.com) - ABCDEFG...
Chave de API 3 (extra.account@gmail.com) - ABCDEFG...
Armazene as chaves assim e faça a rotação.

[IMAGE:04_image_3.png]

Dessa forma, você pode usar sem limites.
Anote-as em um bloco de notas como na imagem abaixo e faça a rotação.
Quanto mais APIs, melhor.

## Solução 2.

Ajustar o consumo de cota da API também é uma estratégia importante.
Apenas reduzindo o escopo da busca, você economiza bastante cota.

[IMAGE:04_image_5.png]
ㅤ
[[purple:Abordagem ineficiente (evitar)]]
- Pesquisar em todos os países
- Definir um escopo desnecessariamente amplo
- Solicitar dados demais de uma vez só
ㅤ
[[purple:Abordagem eficiente (recomendada)]]
- Selecionar apenas os países necessários (apenas os países-alvo como Coreia, EUA, Japão)
- Pesquisar sequencialmente por região
- Solicitar apenas os dados realmente necessários
ㅤ
