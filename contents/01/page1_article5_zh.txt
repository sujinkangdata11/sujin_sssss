就搜索了一次！
却看到了 API 配额超限的错误信息？
我来告诉你原因。

[[purple:搜索的国家越多，API 消耗的配额就越多]]。 
如果在搜索时选择了“全部国家”，
就会向总计 82 个国家发出 API 请求，从而消耗大量配额。

因此，请尝试减少搜索的国家数量。

---

[IMAGE:05_image_2.png]

## 在 Google 控制台里配额还很多，但却报错了。

这是非常常见的问题。
由于 API 配额用量不是实时更新的，
很可能 [[purple:Google 控制台显示的页面尚未反映最新的使用量]]。

---

## 最可靠的解决方法
## 就是使用新的 API 密钥。

为了找到高效的爆款 Shorts，
[[purple: 一定要事先准备多个 API 密钥。]]

[IMAGE:04_image_4.png]

Google 允许创建不限数量的 Gmail 账号，
每个 Gmail 账号都可以申请各自的 API 密钥。
也就是说，你可以 [[purple:按需获取足够多的 API 密钥]]。

立刻用多个 Gmail 账号申请 API 密钥，并整理记录在备忘里。
