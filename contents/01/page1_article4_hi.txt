## "कोटा पार हो गया" एरर का समाधान
ㅤ
API इस्तेमाल करते समय क्या आपको अचानक [[purple:"कोटा पार हो गया"]] जैसा एरर संदेश दिखा?
घबराएँ नहीं।
यह बहुत सामान्य समस्या है, और इसका बेहतरीन समाधान है।
ㅤ
## वर्तमान स्थिति को समझें
वर्तमान में आपकी API कुंजी 24 घंटे की प्रतीक्षा अवस्था में है। लेकिन आपको सिर्फ इंतज़ार ही नहीं करना है!   
ㅤ
[IMAGE:04_image_2.png]

## समाधान 1.
ㅤ
यह सबसे सरल और प्रभावी तरीका है।
Google में साइन-अप पर कोई बड़ा प्रतिबंध नहीं है।
इसलिए आप [[purple:कई Google खाते बनाएँ और प्रत्येक से API कुंजी]] जारी कर सकते हैं।
ㅤ
## व्यावहारिक उपयोग
1. सामान्यतः ही [[purple:3–5 Google खातों से API कुंजियाँ]] पहले से जारी करके रखें
2. [[purple:पहली कुंजी सीमा पर पहुँचते ही दूसरी कुंजी]] पर स्विच करें
3. रोटेशन सिस्टम से बारी-बारी से उपयोग करें
ㅤ
## पहले यथासंभव अधिक API सुरक्षित करें।
ㅤ
API कुंजी 1 (main.account@gmail.com) - ABCDEFG...
API कुंजी 2 (backup.account@gmail.com) - ABCDEFG...
API कुंजी 3 (extra.account@gmail.com) - ABCDEFG...
इस तरह कुंजियाँ सेव रखें और रोटेशन में चलाएँ।

[IMAGE:04_image_3.png]

इस तरह आप बिना सीमा के उपयोग कर पाएँगे।
नीचे की तस्वीर की तरह नोट में लिखकर रखें और रोटेशन करें।
API जितनी अधिक, उतना बेहतर।

## समाधान 2.

API कोटा उपयोग को समायोजित करना भी एक महत्वपूर्ण रणनीति है।
सिर्फ खोज दायरा घटाने भर से, कोटा काफी बचाया जा सकता है।

[IMAGE:04_image_5.png]
ㅤ
[[purple:अप्रभावी तरीका (बचें)]]
- सभी देशों पर सर्च करना
- अनावश्यक रूप से बहुत व्यापक दायरा सेट करना
- एक बार में बहुत अधिक डेटा माँगना
ㅤ
[[purple:प्रभावी तरीका (अनुशंसित)]]
- सिर्फ आवश्यक देशों का चयन (केवल लक्षित देश जैसे कोरिया, अमेरिका, जापान)
- क्षेत्रवार क्रम से सर्च करना
- सिर्फ वही डेटा माँगना जिसकी वास्तव में ज़रूरत हो
ㅤ
